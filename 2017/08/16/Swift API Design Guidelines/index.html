基本规则

  • 在使用的时候清晰度是你的重要目标,实体的方法或声明只有一次,并且反复使用它们.设计API时要保持使用简单明了,一个好的设计仅仅好的声明还够的;经常检查上下文,确保看起来清晰.
  • 清晰比简洁更重要.虽然Swift代码可以紧凑,过分的精简不是我们推荐的目标.精简的Swift语言,在强类型系统和特性中使用是副作用.

  • 给每一个声明写文档注释.通过编写文档的见解,对你良好的设计有深远的影响,请不要忘记.

    如果简单的描述API功能对你来说很麻烦,那么你可能做了错误的设计.
