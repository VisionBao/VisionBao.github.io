探究过程

方法

这里有两种方式，可以让我们了解函数的调用过程：

  • 常规方法，通过单步调试（Step into) 来查看方法的调用过程。这种适用于有源码的情况。
  • 通过查看汇编代码了解调用过程。适用于没有源码的情况。

这里着重介绍一下第二种方式：

  1. 运行代码，将代码的断点设置到上图的位置；
  2. 从菜单栏中点击 Debug -> Debug Workflow -> Always Show Disassembly;
  3. 通过汇编代码的注释，结合单步调试，也可以了解到方法的调用过程。

结论

通过上面两种方式，可以得出如下调用过程：
alloc-> objc_alloc —> callAlloc —> objc_msgSend —> alloc —> _objc_rootAlloc —> callAlloc —>_objc_rootAllocWithZone —> _class_createInstanceFromZone

函数分析

再来看一下函数实现：

objc_alloc

首先调用 objc_alloc 方法，同时 objc_alloc 会调用 callAlloc 方法中，注意此时第二第三参数。

callAlloc

我们注意到流程中出现两个 callAlloc 方法这里详细展开一下

  1. 第一次调用 callAlloc 方法最终会 return 到最后一行return ((id(*)(id, SEL))objc_msgSend)(cls, @selector(alloc));中
  2. 第二次调用 callAlloc 方法最终会 return 到_objc_rootAllocWithZone中

具体可以看一下注释：

alloc

第一次调用 callAlloc 后会调用 alloc 方法，alloc 方法进一步执行_objc_rootAlloc

_objc_rootAlloc

该方法会再一次调用 callAlloc 方法，注意此时的第三个参数为 true(allocWithZone)

_objc_rootAllocWithZone

直接调用_class_createInstanceFromZone 方法

_class_createInstanceFromZone

这里面主要进行三个步骤

  1. 计算需要的大小
  2. 根据大小开辟内存空间
  3. 进行 isa 指针绑定

其他的是一些异常边界处理和 C++扩展的处理

CPU 组成

CPU 内部主要由运算器、控制器、寄存器三大部分组成。

  • 运算器 负责算术运算（+ - * / 基本运算和附加运算）和逻辑运算（包括 移位、逻辑测试或比较两个值等）。
  • 控制器 负责应对所有的信息情况，调度运算器把计算做好。
  • 寄存器 它们可用来暂存指令、数据和地址。既要对接控制器的命令，传达命令给运算器；还要帮运算器记录处理完或者将要处理的数据。
